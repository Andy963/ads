## 2024-03-29 - Missing `aria-label` attributes on interactive components
**Learning:** Found multiple icon-only buttons (`.iconBtn`, `.removeBtn`, `.queue-del`, `.attachmentsClear`, `.attachIcon`, `.stopIcon`, `.sendIcon`, etc) across core interface elements (e.g. `TaskBoard`, `MainChatComposerPanel`, `TaskCreateForm`) that were missing `aria-label` attributes. This prevented screen reader users from understanding the purpose of key actions (like edit, retry, delete, clear images, remove queue item, send message, interrupt).
**Action:** When adding or modifying interactive, icon-only components without visible text labels, explicitly check and enforce the presence of a descriptive `aria-label` attribute, supplementing the standard `title` tooltip for inclusive accessibilty.
