## 2024-05-24 - [Encoded Path Traversal Bypass and Directory Matching]
**Vulnerability:** URL paths were not properly URL-decoded before normalization in the custom HTTP server, allowing path traversal using encoded dots (`%2e`) and slashes (`%2f`). In addition, the `startsWith` boundary check for the static directory allowed partial directory matching (e.g. `distClient-secrets` could match `distClient`).
**Learning:** Node.js's `path.normalize` works on the characters passed. If they are URL-encoded, `path.normalize` treats them as regular file name characters instead of directory separators and parent directory references. Additionally, string matching paths requires appending a path separator to ensure exact base directory constraints.
**Prevention:** Always manually decode URL paths with `decodeURIComponent` (wrapped in a `try-catch` to handle malformed encoding errors) before path normalization. Ensure base directory checks end with `path.sep` to prevent partial name matches.
