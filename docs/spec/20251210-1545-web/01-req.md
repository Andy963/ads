## Introduction
- 背景：Web 控制台已支持多 session，但当前 UI 仅显示“ADS Web Console”标题和单个会话展示，切换/新建/历史入口分散（按钮 + 弹窗），体验接近单会话。
- 目标：在头部采用类似浏览器标签的样式，左侧仅显示“ADS”，右侧以标签方式管理多个 session，提供新建与历史入口的图标化操作。
- 期望成果：用户可直接在头部标签间切换会话，快速新建会话，关闭标签不丢失会话缓存，可通过历史入口恢复。

## Scope
- In Scope：
  - 头部布局改造：单行，左侧文字“ADS”，右侧为会话标签区与图标。
  - 会话标签行为：显示当前及已打开的会话，当前标签高亮，支持关闭。
  - 新建会话入口：使用“+”图标新建并切换。
  - 历史入口：使用“⟳”图标打开历史列表，可恢复已关闭的会话。
  - 前端交互与样式调整，复用现有 session/cache 逻辑。
- Out of Scope：
  - 后端 session 管理协议与 WebSocket 握手逻辑（保持现状）。
  - 规范外的导航、工作流、计划组件改动。
  - 新的权限、审计或国际化改造。

## Functional Requirements

### R1 头部布局与标签区
- 概述：头部单行，左侧仅显示“ADS”；右侧为可滚动的会话标签区，紧随“+”与“⟳”图标。
#### Acceptance Criteria
- [ ] 头部无多行或换行；左侧固定文本“ADS”，不再出现“Web Console”等冗余标题。
- [ ] 会话标签区与图标在右侧同一行展示，标签超出宽度时可水平滚动（不折行）。
- [ ] 原有独立的新建/切换按钮被移除，仅保留标签区、“+”与“⟳”图标。

### R2 会话标签展示与切换
- 概述：标签展示当前打开的 session 列表，点击即切换。
#### Acceptance Criteria
- [ ] 当前 session 标签高亮（颜色/下划线/背景有区分），其他为非激活态。
- [ ] 点击任一非当前标签，会立即切换至对应 session，并恢复该 session 的消息缓存与工作区状态。
- [ ] 标签文本显示 sessionId（或短 ID），支持长 ID 时截断/省略，但可通过 tooltip 或全量文本可见。

### R3 关闭标签行为
- 概述：每个标签包含“✕”，用于从标签区移除。
#### Acceptance Criteria
- [ ] 点击“✕”仅移除标签，不删除本地缓存、不清除 session 历史记录。
- [ ] 若关闭的是当前标签且仍有其他标签，自动切换到最近使用的一个现存标签。
- [ ] 若关闭最后一个标签，自动新建一个 session 并激活，以避免无活动会话。

### R4 新建会话（“+”图标）
- 概述：右侧“+”图标用于新建并激活 session。
#### Acceptance Criteria
- [ ] 点击“+”生成新的 sessionId，清空当前消息展示区域，载入该 session 的历史缓存（若已存在）并建立新的 WebSocket 连接。
- [ ] 新会话自动加入标签区并高亮，记入 session 历史列表。
- [ ] 新建操作不影响其他已打开标签的状态。

### R5 历史入口（“⟳”图标）
- 概述：历史图标打开历史列表以恢复会话。
#### Acceptance Criteria
- [ ] 点击“⟳”打开历史列表（可沿用现有历史弹窗/侧栏逻辑），展示已记录的 session 历史（沿用现有 15 条上限）。
- [ ] 从历史中选择一个 session 时：在不重复的情况下将其添加为新标签并切换，保留其缓存；若已存在标签则仅切换。
- [ ] 历史列表空态有明确提示，不影响标签区正常操作。

### R6 状态持久与一致性
- 概述：标签区与历史列表基于现有本地存储（SESSION_HISTORY_KEY 等），保持数据一致。
#### Acceptance Criteria
- [ ] 页面刷新后，当前活跃 session 与打开的标签集可从本地历史恢复，至少包含上次活跃的 session。
- [ ] 不因标签增删影响现有的 per-session 缓存、工作目录、token 使用逻辑。
- [ ] 不新增后端存储字段或协议变更。

## Non-Functional Requirements
- 兼容性：保持现有深浅色方案；不引入额外的阻塞加载或显著性能回退。
- 可用性：标签点击/关闭/新建/历史弹窗均提供 hover/aria-label 提示；交互在网络断开/重连场景下不崩溃。

## Observability & Alerting
- 保持现有日志与前端状态提示；不新增后端监控指标需求。

## Release & Timeline
- 开发与测试：待设计/实施计划明确后排期。
- 回滚方案：前端可通过特性开关或回退到单会话头部布局（需在实施计划中明确）。
