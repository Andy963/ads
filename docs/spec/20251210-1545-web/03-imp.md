## 任务拆解

- [ ] T1 头部结构与样式调整  
  - 负责人：未指定  
  - Steps：  
    - 将 header 改为单行：左侧仅 “ADS”+ ws 状态灯；右侧容纳标签区与图标。  
    - 移除旧“新建/切换”按钮，添加标签容器、图标容器。  
    - 标签区支持水平滚动；长 ID 省略，保留 title 提示。  
  - Verification：  
    - 头部不换行；“ADS”始终在左，标签区在右。  
    - 溢出时可滚动，无布局断裂。

- [ ] T2 会话标签渲染与切换  
  - Steps：  
    - 基于打开会话列表渲染标签，当前高亮。  
    - 点击标签：切换 currentSessionId，恢复对应缓存，重连 ws（携带 ads-session）。  
    - 关闭标签：仅移除标签，不清缓存；关闭当前则切到最近标签；无剩余时自动新建。  
  - Verification：  
    - 多标签切换后消息缓存对应；关闭当前后自动切换或新建生效。  
    - 不出现重复重连提示（4409 抑制正常）。

- [ ] T3 “+”新建与“⟳”历史入口  
  - Steps：  
    - “+”生成新 sessionId，追加标签并激活，记入历史，恢复缓存并重连。  
    - “⟳”打开历史列表（复用现有弹窗），选择项若不存在标签则追加并激活，存在则仅切换。  
    - 历史空态提示友好。  
  - Verification：  
    - “+”后能看到新标签且成为当前；历史列表可恢复关闭的 session。  
    - 选择历史中已有标签时不重复创建。

- [ ] T4 状态存储与一致性  
  - Steps：  
    - 复用 SESSION_KEY、SESSION_HISTORY_KEY、per-session cacheKey；刷新后恢复上次活跃 session 与标签集合（至少当前标签）。  
    - 不改后端协议、token 处理；处理本地存取异常的兜底。  
  - Verification：  
    - 刷新后仍在上次 session；关闭标签后历史可找回；缓存未错乱。  
    - ws 子协议仍包含 ads-session，未引入新字段。

- [ ] T5 验证与回归  
  - Steps：  
    - 手动：多标签切换/关闭/新建/历史恢复，长 ID 省略、滚动、断网重连（4409 抑制）、刷新恢复。  
    - 运行 `npm test`，记录已有失败（若仍存在，注明与本改动无关）。  
  - Verification：  
    - 手动场景通过；`npm test` 结果记录，确认无新增失败。

## 风险与缓解
- 已有测试历史失败可能遮挡新增问题 → 跑测试并比对失败集，必要时针对相关模块补充自测。  
- 标签关闭/切换逻辑引发状态错乱 → 保持唯一 currentSessionId 来源，操作前后同步缓存/历史。  
- UI 溢出或无标签态 → 默认创建新 session，标签区可滚动。
