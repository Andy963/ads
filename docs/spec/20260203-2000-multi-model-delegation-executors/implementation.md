# 多模型指派（一次性子执行器）- 实现备忘（暂缓）

> 说明：本文件仅用于把讨论结论落盘，便于未来实现；当前不实现、不承诺时间表。

## 1. 目标回顾（仅一页纸）
- 主会话固定模型（例如 `model-primary`）且不被指派流程影响。
- 每次指派创建一次性子执行器，指定 `model`（例如 `model-fast`），执行完销毁对话历史。
- 每次指派都记录实际使用的 `model`，可用于审计与统计。
- 同项目内写入串行（单写者），并发只用于分析/产出建议或 patch。

## 2. 未来实现的关键点（Checklist）
- 任务卡 schema 增加 `model` 字段，并在本地任务记录中持久化。
- 指派执行路径支持“按次指定模型”：
  - 创建新的子执行器对话线程（no history reuse）。
  - 执行多轮工具调用（如需要），但该执行器生命周期仅限本次指派。
- 写入策略落地：
  - 默认让子任务产出 patch，由主会话串行应用。
  - 或至少保证写入动作经过同一把写锁。
- 冲突处理流程：
  - apply 失败 -> 打回重生成 / 主会话合并 / 放弃。
- 可观测性：
  - UI/日志输出包含 `model`。
  - 本地记录可查询到 `model`。

## 3. 建议的验收用例（未来）
- 主会话连续对话保持 thread 不变；指派执行不影响主会话上下文。
- 连续两次指派使用同一模型，但互不共享上一次指派上下文（输出不带入旧任务细节）。
- 两个子任务建议改同一文件：
  - 主会话能检测 apply 冲突并触发打回策略。
  - 写入过程不会出现互相覆盖的“脏写”。
