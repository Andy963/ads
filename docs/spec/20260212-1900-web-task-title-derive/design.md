# 设计：前端派生标题（UX）+ 后端兜底（正确性）

## 标题派生规则（与后端创建保持一致）

给定 `prompt`：

1. 取第一条非空行：按 `\n` 分割后对每行 `trim()`，取第一个长度 > 0 的行。
2. 压缩空白：用正则 `\s+` 替换为单个空格。
3. 截断到 32 字符并追加省略号 `…`。
4. 若找不到非空行，使用默认标题 `新任务`。

## 前端（TaskBoard 编辑）

- 保存时先校验 `prompt` 非空。
- 若 `editTitle.trim()` 为空：
  - 按上述规则从 `prompt` 派生标题
  - 将派生值写回 `editTitle`（便于后续校验失败时用户看到回填结果）
  - 用派生值继续提交更新

创建任务保持现状：发送 `title: undefined` 交由后端生成，避免重复实现导致规则漂移。

## 后端（TaskStore.updateTask 兜底）

- 将标题派生逻辑抽取为内部函数，并复用于 `createTask()`。
- 在 `updateTask()` 中增加兜底：
  - 若 `title` 为空且 `prompt` 非空，则派生标题并写入。
  - 若 `title` 与 `prompt` 均为空，则保留原 `title`（避免意外覆盖为默认值）。

## 测试策略

- 后端：增加 `TaskStore.updateTask()` 的单测覆盖“更新空标题时派生标题”和“标题+prompt 都为空时保留原值”。
- 前端：增加 `TaskBoard` 编辑 modal 的组件测试，覆盖“标题为空时派生并提交”。
