# 需求：编辑任务时自动派生标题

## 背景

当前创建任务时允许不填标题：后端会从 `prompt` 派生一个标题用于展示与检索。

但在前端编辑任务时，`TaskBoard` 的编辑表单会强制要求标题非空，导致“无标题任务”在编辑 `prompt` 时必须手动补标题才能保存，体验与创建行为不一致。

同时，部分前端任务状态标签与 badge 未覆盖 `queued` / `paused`，会出现英文原始状态字符串，影响 UI 一致性。

## 目标

- 编辑任务时：若标题为空，前端自动从 `prompt` 派生标题并回填，再提交更新，保持 UX 一致性。
- 后端继续兜底：防止客户端遗漏或规则漂移导致持久化空标题。
- UI 文案一致性：补齐 `queued` / `paused` 的展示标签与 badge。

## 需求范围

- 前端：
  - `TaskBoard` 编辑保存：空标题自动派生并提交。
  - `TaskBoard` 状态标签：补齐 `queued` / `paused` 中文标签。
  - `TaskList` badge：补齐 `paused` 的大写 badge。
- 后端：
  - `updateTask()`：若 `title` 为空且 `prompt` 非空，则按创建同样的规则派生标题（兜底）。

## 非目标

- 不强制让“创建任务”在前端也派生标题并发送（避免与后端规则漂移）。
- 不引入新的全局 i18n 框架或文案系统。

## 验收标准

- 编辑任务时将标题清空仍可保存，且更新请求提交的 `title` 为派生值。
- 后端不会把空标题写入数据库（在更新路径上兜底生效）。
- `queued` / `paused` 状态在任务面板显示为中文标签，`paused` 在任务列表显示为 `PAUSED`。
