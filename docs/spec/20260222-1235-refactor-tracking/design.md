# Design

## 总体策略

采用“长期追踪 + 小步迭代”的重构流程：

1. **追踪先行**：建立 `docs/REFACTOR.md`，先记录已读与未读范围，避免无序扩散。
2. **局部重构**：选择低风险、可验证的重构点（例如去重小工具函数、收敛散落的解析逻辑）。
3. **验证闭环**：每次迭代在本地跑 `tsc + lint + test`，确保行为不回退。
4. **PR 粒度控制**：每个 PR 聚焦一类重构点，避免混杂“重构 + 功能改动”。

## `docs/REFACTOR.md` 结构

- **Reviewed Modules**：已详细阅读的模块（文件/目录），并附上阅读意图（入口/热路径/边界条件）。
- **Candidates / Opportunities**：按维度（结构/可扩展性/可维护性/性能）列出候选，允许标注 DONE。
- **Not Yet Reviewed**：未阅读模块列表，便于后续系统化推进。

## 风险与控制

- 风险：重构引入隐性行为变更。
  - 控制：限制重构粒度；优先提取纯函数；对关键路径补测试（若现有覆盖不足）。
- 风险：文档无法提交（被 `.gitignore` 忽略）。
  - 控制：显式在 `.gitignore` 中放行 `docs/REFACTOR.md`。
