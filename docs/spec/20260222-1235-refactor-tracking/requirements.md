# Requirements

## 背景

当前 ADS 代码库功能快速演进，模块边界与工具函数存在一定程度的重复与耦合；同时，重构活动需要可追踪、可回滚、可验收，避免“重构到一半不知道改了什么/看过哪里”。

## 目标

- 提供一个长期维护的重构追踪文档：`docs/REFACTOR.md`，用于记录：
  - 已阅读过的模块范围（至少到文件/目录粒度）。
  - 已识别的重构点（结构、可扩展性、可维护性、性能）。
  - 仍未阅读的模块清单（避免遗漏）。
- 重构方式遵循“Slow is Fast”：
  - 小步、可审阅、可回滚（优先以小 PR 迭代）。
  - 不改变对外行为与现有功能。
  - 保持测试绿（本地校验通过后再提交）。

## 非目标

- 不做大规模架构重写（例如一次性拆分服务或引入全新框架）。
- 不引入新的外部依赖（除非明确收益且必要）。
- 不为了“抽象统一”牺牲清晰度（避免过度工程）。

## 约束

- 不影响现有测试与功能。
- 修改后需通过：

```bash
npx tsc --noEmit
npm run lint
npm test
```

- 若修改前端（`web/`）相关代码，额外通过：

```bash
npm run build
```

## 验收标准

- `docs/REFACTOR.md` 被纳入版本控制（不被 `.gitignore` 排除）。
- 每次重构迭代至少更新一次 `docs/REFACTOR.md`（新增“已读模块/重构点/未读模块”的变更之一）。
- 新增/调整的重构代码通过类型检查、lint、测试。
